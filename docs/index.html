ScriptuccinoJS

v2.0.0-alpha

View on GitHub

a simple set of JavaScript function and utility modules for easily performing some common tasks

Installation

ScriptuccinoJS is available as a set of ES6 modules, as well as a UMD library.

NPM

The recommened way to install ScriptuccinoJS is via the NPM package manager:

npm install --save @beyondthesketch/scriptuccinojs

CDN

Alternatively, the files are available via CDN:


<!-- ES Modules -->
<script
    type="module"
    src="https://unpkg.com/@beyondthesketch/scriptuccinojs@1.8.1/index.js"
></script>

<!-- UMD ES5 Library -->
<script
    src="https://unpkg.com/@beyondthesketch/scriptuccinojs@1.8.1/scriptuccino.js"
></script>

UNPKG is a common choice, but as scriptuccino is published as an NPM package, other CDNs that host those packages could also be used, such as JSdelivr

Download

If you prefer, you can download the files and host them directly:

The ES6 modules are in the esm, directory and the ES5 UMD library is in the umd directory.

Usage

The following examples assume that the ScriptuccinoJS modules and library script are being served from a directory named /scriptuccino/.

ES modules

Import the modules you need into your script from their respective files in the esm/ folder:

Use static import:


import whenPageLoaded from '/scriptuccino/esm/whenPageLoaded.js';

const myFoo = () => alert('foo');

whenPageLoaded(myFoo);

Using dynamic import:


const myFoo = () => alert('foo');

import('/scriptuccino/esm/whenPageLoaded.js')
    .then(
        ({ default: whenPageLoaded }) => whenPageLoaded(myFoo)
    );

Using dynamic import with await:


const myFoo = () => alert('foo');

const { default: whenPageLoaded } = await import('/scriptuccino/esm/whenPageLoaded.js');

whenPageLoaded(myFoo);

ES5 Library

ScriptuccinoJS is also provided as a ES5 library, in a UMD file. Add the file to your webpage (found in the umd/ folder), preferably at the bottom of the body - but above your own scripts (external or inline):


<!-- -->
<body>
<h1>Foo</h1>
<p>Lorem ipsum dolor sit amet...</p>

<script src="/scriptuccino/umd/scriptuccino.min.js"></script>
<script src="/your.own.script.js"></script>
</body>
<!-- -->

In your own scripts, use the ScriptuccinoJS functions you need using the scriptuccino global object:


const myFoo = () => alert('foo');

scriptuccino.whenPageLoaded(myFoo);

API

Modules

ScriptuccinoJS is organised into the following modules, each containing a number of utilities. If using ScriptuccinoJS as ES modules you can import them from any of the files they are documented as being exported from below. If using the ES5 library, you can access them from the scriptuccino global/namespace object as described.

events

Utilities and functions for performing operations related to events, such as page load.

whenPageLoaded( fn: Function )

Queue the supplied function for execution once the page is loaded (executes immediately if page is already loaded).

whenPageReady( fn: Function )

Queue the supplied function for execution once the DOM is ready (executes immediately if the DOM is already ready).

whenScrollEnds( fn: Function )

Run the supplied function when the window scrolling comes to a stop. The scroll detection is debounced to 250 milliseconds.

utils

Tools for doing common things with a little more ease.

elementComesIntoViewport( elements: Element | Nodelist, fn: Function[, (outFn?: Function), options: Object ] )

uses: whenScrollEnds (as fallback)

WARNING: This utility uses experimental web features (Intersection Observer) which is not yet finalised and could change. IT MAY BREAK/THROW ERRORS if the user agent uses a different implementation.

It falls back to whenScrollEnds functionality if Intersection Observer is not supported at all. Note that this fallback implementation only considers movement through the viewport vertically and not horizontally.

While this function's purpose is to detect when an element enters the viewport, it is also often useful to detect when that element exits too. As a result, this function has 2 forms of use:

  1. Call the supplied function as soon as the specified element is scrolled into the viewport using the IntersectionObserver API.

    This form takes one callback function and an optional options object for configuring the observer instance.

    
elementComesIntoViewport(
    document.getElementById('foo'),
    (element, observer) => {
        // do something
    },
    {
        threshold: 0.25
    }
);

  2. Call the first supplied function as soon as the specified element is scrolled into the viewport, and call the second supplied function as soon as the element is scrolled out of the viewport, using the IntersectionObserver API.

    The second form takes two callback functions and an optional options object for configuring the observer instance.

    
elementComesIntoViewport(
    document.getElementById('foo'),
    (element, observer) => {
        // do something on entering viewport
    },
    (element, observer) => {
        // do something on leaving viewport
    },
    {
        threshold: 0.25
    }
);

The callback functions fn and outFn both receive 2 arguments, the first is the element being observed and the second is the IntersectionObserver instance.

The options argument is a plain object to customise the observer that is created. Please see IntersectionObserver API documents for details on what properties this can take.

There is only one IntersectionObserver instance created for all nodes that are passed to this utility. The element will continue to trigger intersections unless you unobserve it in the supplied function.

elementIsAboveFold( element: Element )

Return `true` if the supplied element is currently above the page fold/viewport, otherwise returns `false`.

elementScrolledAboveFold( elements: Element | Nodelist, fn: Function )

uses: whenScrollEnds

Call the supplied function when the window is scrolled so that the specified element or elements are above the page fold.

The callback function fn receives a single argument which is the element that has settled above the fold.

XHR( config: Object )

AJAX (or XMLHttpRequest as it is officially named) is AWESOME! But it's a little fiddly to setup. Use this util to quickly configure and optionally send them.

lazyLoadCSS( uri: string [, callbackFn: Function] )

uses: XHR, whenPageLoaded

Download (asynchronously with AJAX) and apply a stylesheet to the page, and optionally execute a callback function, only once the page has loaded.

parallelLoadCSS( uri: string [, callbackFn: Function] )

uses: XHR, whenPageReady

Download (asynchronously with AJAX) and apply a stylesheet to the page, and optionally execute a callback function, when the DOM is ready. Useful for deferred load of CSS on larger pages with longer load times.

simpleDebounce( fn: Function, delay: Number )

Perform the supplied function only if it is not called again within the timeframe specified by delay, in milliseconds

fx

Programmatically apply CSS transitions. Great for throw-away transitions, prototyping or quickly applying complex effects without having to figure out the CSS.

applyTransition( element: HTMLElement, settings: Object|Array, styles: Object [, completeFn: Function] )

Define and apply CSS transition to an HTML element, and optionally perform a callback function when the transition is complete.

The optional settings argument takes either a plain object or an array of objects which determine the specification of the transition/s.

The properties and types of this object are:


{
    property: String,
    duration: Number,
    curve: String,
    delay: Number,
}

The number values represent milliseconds. The property property is the CSS property to transition, the curve is the timing function to use, which must be provided as a string.

fadeIn( element: HTMLElement [, completeFn: Function, settings: Object] )

uses: applyTransition

Apply a fade-in transition to an HTML element. Optionally perform a callback when complete and define optional settings for the transition.

Default duration is 500ms. Default timing function is ease. The settings object is the same as for applyTransition.

The settings object supports any of the relevant transitions property except property, which is predefined - if supplied in the object, it will simply be ignored.
